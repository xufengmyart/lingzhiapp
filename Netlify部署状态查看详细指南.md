# Netlify Dashboard 部署状态查看详细指南

## 第一步：访问 Netlify Dashboard

### 打开浏览器访问
https://app.netlify.com

### 登录您的账号
- 如果未登录，点击右上角 **"Sign in"**
- 使用 GitHub 登录（与注册时相同）
- 输入用户名和密码，点击 **"Sign in"**

---

### 进入团队页面
登录后会跳转到 **"Your team"** 页面

### 查看项目列表
页面上会显示您的所有项目，格式如下：

```

|   项目名称         |   状态            |
+-------------------+-------------------+
| lingzhiapp        | ✓ Published       |
+-------------------+-------------------+
| (其他项目...)      |                   |
+-------------------+-------------------+
```

### 进入项目
点击 **"lingzhiapp"** 项目卡片

## 第三步：查看部署进度

### 进入项目概览
点击后会进入 **"Site overview"** 页面

### 查看顶部状态栏
页面顶部会显示：

```
🚀 lingzhiapp
🔗 https://xxxxx.netlify.app
✓ Published master@d8398ee
```

### 点击 "Deploys" 标签
在页面导航栏中找到并点击 **"Deploys"**

### 部署列表页面
页面会显示所有部署记录，从新到旧排序：

```
+--------+----------------+------------------+------------------+
| 状态   | 分支/提交ID     | 时间              | 域名             |
+--------+----------------+------------------+------------------+
| ✓ Pub  | master@d8398ee | Just now          | xxxxx.netlify.app|
| ⏳ Proc | master@4c5bd8a | 2 hours ago       |                  |
| ✓ Pub  | main@36ae612   | 3 hours ago       |                  |
+--------+----------------+------------------+------------------+
```

### 状态图标含义

| 图标 | 含义 | 说明 |
|------|------|------|
| ✓ | Published | 部署成功，已上线 |
| ⏳ | Processing | 正在部署中 |
| ❌ | Failed | 部署失败 |
| ⏸ | Canceled | 已取消 |

### 点击最新部署
点击顶部的 **"✓ Published master@d8398ee"**

1. **顶部导航栏**

   - 菜单按钮

2. **主页面**

   - 用户头像/登录按钮

3. **功能模块**

## 常见部署问题

### 问题 1：部署失败 (❌ Failed)

**症状**：Deploy 列表显示红色 ❌

**原因**：代码错误、依赖问题

**解决**：
1. 点击失败的部署
2. 查看构建日志中的错误信息
3. 根据错误信息修复代码
4. 推送代码后自动重新部署

### 问题 2：部署中 (⏳ Processing) 超过 5 分钟

**症状**：状态一直显示 ⏳

**原因**：构建卡住、网络问题

**解决**：
1. 等待更长时间（最多 10 分钟）
2. 刷新页面
3. 如果仍然卡住，点击 **"Cancel deploy"**
4. 推送新代码触发重新部署

### 问题 3：部署成功但页面空白

**症状**：✓ Published 但访问显示空白

**原因**：
- JavaScript 错误
- 路由配置问题

**解决**：
1. 按 F12 打开开发者工具
2. 查看 **Console** 标签的错误信息
3. 查看 **Network** 标签，看哪些请求失败

### 问题 4：显示旧版本

**症状**：部署成功但显示的是旧内容

**原因**：浏览器缓存

**解决**：
1. 按 **Ctrl + Shift + R** 强制刷新
2. 清除浏览器缓存
3. 隐私模式访问

## 自动部署说明

### 触发条件
Netlify 会在以下情况自动部署：

1. **推送到 GitHub** - 任何推送到 `main` 或 `master` 分支的代码
2. **Pull Request** - 任何新的 Pull Request
3. **手动触发** - 在 Dashboard 点击 **"Trigger deploy"**

### 部署流程
```
推送代码 → GitHub → Netlify Webhook → 触发构建 → 部署 → 上线
```

### 查看部署历史
在 **"Deploys"** 标签可以看到所有历史部署
