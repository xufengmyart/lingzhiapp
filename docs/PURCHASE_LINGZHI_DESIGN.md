## 设计概述

### 核心理念

购买灵值不是简单的"花钱买灵值"，而是**文化价值投资**和**生态加速器**。用户购买灵值的同时，是在：

1. **投资文化传承**：支持西安文化保护和转译项目
2. **加速生态参与**：快速获得灵值，参与高级别项目和活动
3. **解锁专属权益**：获得普通用户无法享受的高端服务和体验
4. **数字资产增值**：购买灵值与贡献值同等价值，可锁定增值

### 设计原则

1. **价值匹配**：购买灵值的价值必须与获得的权益对等
2. **激励贡献**：不降低用户通过贡献获得灵值的动力
3. **生态平衡**：购买灵值占比控制在合理范围（建议不超过30%）
4. **透明公平**：所有购买规则、权益、使用场景公开透明
5. **文化导向**：购买灵值主要用于文化项目和服务，而非简单消费

### 目标用户

1. **时间稀缺型用户**：没有足够时间参与任务，但想快速获得灵值
2. **项目需求型用户**：需要大量灵值启动文化转译项目
3. **体验优先型用户**：希望享受高级别、高品质的文化服务
4. **投资型用户**：将灵值视为数字资产，进行投资增值

## 价值主张

### 用户视角

**我为什么要购买灵值？**

1. **快速启动项目**：需要1000灵值启动文化转译项目，通过任务需要1个月，购买1天即可
2. **享受高端服务**：购买灵值后可以预约顶级文化专家进行1对1咨询
3. **解锁专属活动**：参与"长安夜宴"、"终南禅修"等线下高端文化体验
4. **优先权益**：美学侦探任务、新项目优先参与权
5. **数字资产投资**：购买的灵值可锁定增值，预期收益20%-100%

### 生态视角

**为什么生态需要购买灵值机制？**

1. **资金来源**：为文化转译项目、生态建设提供资金支持
2. **生态加速**：快速积累生态价值，加速双螺旋飞轮运转
3. **用户分层**：通过购买机制实现用户分层，提供差异化服务
4. **价值锚定**：购买行为强化灵值的现实价值锚定（1灵值=0.1元）

### 限时优惠档位

**首充双倍**（限首次充值）
- 充值任意档位，额外赠送50%灵值
- 例如：购买新手体验包（99元），获得1100灵值 + 550灵值 = 1650灵值

**节日特惠**（重大节日）
- 节日期间特定档位，额外赠送30%灵值

**批量购买**（一次性购买多个档位）
- 购买3个档位：额外赠送20%
- 购买5个档位：额外赠送30%
- 购买10个档位：额外赠送50%

## 购买权益

### 权益分类

#### 1. 服务类权益

**线上文化沙龙**
- 内容：西安文化主题线上讲座
- 频率：每周1次
- 价值：50元/次
- 权益：根据档位获得免费次数

**文化专家1对1咨询**
- 内容：专业文化顾问1对1咨询
- 时长：30分钟/次
- 价值：300元/次
- 权益：根据档位获得免费次数

**品牌空间设计服务**
- 内容：专业设计师提供品牌空间设计方案
- 价值：2000-5000元/项目
- 权益：根据档位获得优惠券

#### 2. 优先权类权益

**美学侦探任务优先权**
- 内容：美学侦探任务优先获得权
- 权益：根据档位获得优先次数
- 价值：更快获得优质任务，获得更多灵值

**新项目优先参与权**
- 内容：新发布的文化转译项目优先参与权
- 权益：合伙人专属
- 价值：参与高价值项目，获得更高收益

#### 3. 访问权类权益

**文化转译案例库访问权**
- 内容：访问平台所有文化转译成功案例
- 价值：学习优秀案例，提升自己的转译能力
- 权益：根据档位获得访问时长

#### 4. 活动类权益

**线下文化体验活动**
- 内容："长安夜宴"、"终南禅修"等线下高端文化体验
- 价值：2000-5000元/次
- 权益：合伙人优先参与权

**年度文化盛典**
- 内容：灵值生态年度盛典活动
- 价值：VIP门票价值1000元/张
- 权益：根据档位获得门票数量

#### 5. 合伙人类权益

**合伙人资格**
- 内容：直接获得对应级别的合伙人资格
- 价值：无需累计灵值，立即享受合伙人权益

**合伙人专属服务**
- 内容：专属咨询、专属社群、专属客户经理
- 价值：享受高品质专属服务

**公司股权期权**
- 内容：获得公司股权期权授予资格
- 价值：未来股权增值收益

**文化项目投资优先权**
- 内容：优先参与平台孵化的文化项目投资
- 价值：项目投资回报

- **灵值消耗**：1000-10000灵值

  - 空间设计（唐风茶馆、餐厅、民宿）

## 技术实现方案

### 数据库设计

#### 新增表结构

**充值档位表 (recharge_tiers)**

```sql
CREATE TABLE recharge_tiers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,                          -- 档位名称
    description TEXT,                            -- 档位描述
    price DECIMAL(10,2) NOT NULL,                -- 价格（元）
    base_lingzhi INTEGER NOT NULL,               -- 基础灵值
    bonus_lingzhi INTEGER NOT NULL,              -- 赠送灵值
    bonus_percentage INTEGER NOT NULL,           -- 赠送比例
    partner_level INTEGER DEFAULT 0,             -- 对应合伙人级别（0-4）
    benefits TEXT,                               -- 权益JSON
    status TEXT DEFAULT 'active',                -- 状态
    sort_order INTEGER DEFAULT 0,                -- 排序
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**充值记录表 (recharge_records)**

```sql
CREATE TABLE recharge_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,                    -- 用户ID
    tier_id INTEGER NOT NULL,                    -- 档位ID
    order_no TEXT UNIQUE NOT NULL,               -- 订单号
    amount DECIMAL(10,2) NOT NULL,               -- 充值金额
    base_lingzhi INTEGER NOT NULL,               -- 基础灵值
    bonus_lingzhi INTEGER NOT NULL,              -- 赠送灵值
    total_lingzhi INTEGER NOT NULL,              -- 总灵值
    payment_method TEXT,                         -- 支付方式
    payment_status TEXT DEFAULT 'pending',        -- 支付状态
    payment_time TIMESTAMP,                      -- 支付时间
    transaction_id TEXT,                         -- 支付交易号
    status TEXT DEFAULT 'active',                -- 状态
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (tier_id) REFERENCES recharge_tiers(id)
);
```

**灵值消费记录表 (lingzhi_consumption_records)**

```sql
CREATE TABLE lingzhi_consumption_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,                    -- 用户ID
    consumption_type TEXT NOT NULL,              -- 消费类型
    consumption_item TEXT NOT NULL,              -- 消费项目
    lingzhi_amount INTEGER NOT NULL,             -- 灵值数量
    item_id INTEGER,                             -- 项目ID（可选）
    description TEXT,                            -- 描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**用户权益表 (user_benefits)**

```sql
CREATE TABLE user_benefits (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,                    -- 用户ID
    benefit_type TEXT NOT NULL,                  -- 权益类型
    benefit_name TEXT NOT NULL,                  -- 权益名称
    benefit_count INTEGER DEFAULT 0,             -- 权益数量（次数）
    benefit_expiry TIMESTAMP,                    -- 权益过期时间
    source TEXT,                                 -- 来源（充值/合伙人）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
