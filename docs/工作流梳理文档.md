## 1. 私有资源管理工作流

### 流程概述
私有资源是用户的核心资产，用于存储和管理个人的资源信息，包括政府资源、企业资源、人脉资源等。

### 工作流步骤

#### 1.1 创建私有资源
```
用户操作流程：
1. 进入"智能助手" → "私有资源" → "资源管理"
2. 点击"添加资源"按钮
3. 填写资源信息：
   - 资源名称（必填）
   - 资源类型：政府/企业/人脉/其他
   - 所属部门
   - 联系人信息（姓名、电话、邮箱）
   - 职位
   - 资源描述
   - 能解决的问题
   - 风险等级：低/中/高
   - 可见性：私有/可匹配
4. 提交保存

系统处理：
- 验证必填字段
- 检查联系方式格式
- 设置默认状态：未授权、待验证
- 保存到数据库
```

#### 1.2 查看私有资源
```
用户操作流程：
1. 进入"私有资源" → "资源管理"
2. 使用筛选器：
   - 授权状态：全部/未授权/已授权/待授权
   - 资源类型：全部/政府/企业/人脉/其他
   - 搜索关键词
3. 查看资源列表，包含：
   - 资源名称和类型
   - 联系人信息
   - 授权状态（图标标识）
   - 风险等级（颜色标识）
   - 验证状态
   - 创建时间
```

#### 1.3 编辑私有资源
```
用户操作流程：
1. 在资源列表中找到目标资源
2. 点击"编辑"按钮
3. 修改资源信息
4. 提交保存

系统处理：
- 验证数据完整性
- 更新数据库记录
- 记录更新时间
```

#### 1.4 删除私有资源
```
用户操作流程：
1. 在资源列表中找到目标资源
2. 点击"删除"按钮
3. 确认删除

系统处理：
- 软删除（设置deleted_at时间戳）
- 不物理删除，保留历史记录
```

#### 1.5 设置资源可见性
```
私有：仅自己可见，不参与匹配
可匹配：参与智能匹配系统，可以被其他项目推荐

切换流程：
1. 编辑资源
2. 修改"可见性"字段
3. 保存
```

## 2. 项目推荐工作流

### 流程概述
项目推荐系统基于用户的私有资源，智能匹配合适的项目，帮助用户发现机会。

### 工作流步骤

#### 2.1 获取推荐项目
```
用户操作流程：
1. 进入"智能助手" → "私有资源" → "项目推荐"
2. 切换到"推荐项目"标签页
3. 查看推荐的项目列表，包含：
   - 项目名称和描述
   - 匹配度评分（0-100）
   - 预算金额
   - 参与费用
   - 项目状态
   - 参与人数

系统处理：
- 扫描用户的所有"可匹配"私有资源
- 与项目库中的需求进行匹配
- 计算匹配度（基于技能、资源类型、地理位置等）
- 返回按匹配度排序的项目列表
```

#### 2.2 申请参与项目
```
用户操作流程：
1. 在推荐项目列表中找到感兴趣的项目
2. 点击"申请参与"按钮
3. 确认参与费用
4. 提交申请

系统处理：
- 检查用户灵值余额是否足够支付参与费用
- 扣除参与费用
- 创建项目参与记录
- 通知项目创建人
- 更新项目参与人数
```

#### 2.3 查看匹配详情
```
用户操作流程：
1. 在推荐项目列表中点击项目
2. 查看匹配详情：
   - 匹配资源列表
   - 匹配原因说明
   - 匹配度评分依据

系统处理：
- 显示具体哪些资源匹配了项目需求
- 解释匹配逻辑和评分规则
```

#### 2.4 资源匹配管理
```
切换到"资源匹配"标签页：
1. 查看所有资源匹配记录
2. 筛选匹配状态：待处理/已接受/已拒绝
3. 对匹配记录进行操作：
   - 接受匹配
   - 拒绝匹配
   - 查看详情
```

## 3. 项目流程管理工作流

### 流程概述
项目流程管理用于跟踪项目的里程碑和任务，确保项目按计划推进。

### 工作流步骤

#### 3.1 选择项目
```
用户操作流程：
1. 进入"智能助手" → "私有资源" → "项目流程"
2. 从项目列表中选择要管理的项目
3. 查看项目的里程碑和任务概览
```

#### 3.2 查看里程碑
```
里程碑状态说明：
- 待完成（灰色）：尚未开始
- 进行中（蓝色）：正在执行
- 已完成（绿色）：已完成
- 已延期（红色）：超过计划完成时间

用户操作流程：
1. 展开里程碑查看详情
2. 查看里程碑的进度百分比
3. 查看里程碑的任务列表
```

#### 3.3 管理任务
```
创建任务：
1. 选择里程碑
2. 点击"添加任务"按钮
3. 填写任务信息：
   - 任务名称
   - 任务描述
   - 负责人
   - 优先级：低/中/高/紧急
   - 预估工时
   - 截止日期
4. 提交创建

更新任务状态：
1. 在任务列表中找到任务
2. 更新任务状态：
   - 待处理
   - 进行中
   - 已完成
   - 已阻塞
3. 更新实际工时
4. 提交更新
```

#### 3.4 监控项目进度
```
项目进度展示：
- 项目整体进度百分比
- 已完成/进行中/待处理里程碑数量
- 任务完成率
- 延期风险提示

用户操作流程：
1. 查看项目概览仪表盘
2. 分析进度数据
3. 识别风险和瓶颈
4. 采取相应措施
```

## 4. 用户资源匹配工作流

### 流程概述
用户资源匹配系统自动将用户的资源与项目需求进行智能匹配，实现资源价值最大化。

### 工作流步骤

#### 4.1 触发自动匹配
```
系统自动执行：
1. 定时扫描所有"可匹配"状态的私有资源
2. 扫描项目库中的开放项目
3. 基于以下维度进行匹配：
   - 资源类型与项目需求类型
   - 技能关键词匹配
   - 地理位置匹配
   - 资源估值与项目预算
   - 风险等级评估
4. 计算匹配度评分
5. 创建匹配记录

用户手动触发：
1. 进入"私有资源" → "资源管理"
2. 点击"自动匹配"按钮
3. 系统立即执行匹配
4. 显示匹配结果数量
```

#### 4.2 查看匹配结果
```
用户操作流程：
1. 进入"私有资源" → "项目推荐" → "资源匹配"标签页
2. 查看匹配记录列表，包含：
   - 匹配资源名称
   - 匹配项目名称
   - 匹配度评分
   - 匹配原因说明
   - 匹配状态：待处理/已接受/已拒绝
   - 创建时间
```

#### 4.3 处理匹配记录
```
接受匹配：
1. 点击"接受"按钮
2. 系统自动创建项目参与申请
3. 更新匹配状态为"已接受"

拒绝匹配：
1. 点击"拒绝"按钮
2. 可选择拒绝原因
3. 更新匹配状态为"已拒绝"
4. 系统记录拒绝原因用于优化匹配算法
```

#### 4.4 匹配优化
```
系统持续优化：
- 记录用户接受/拒绝行为
- 分析匹配模式的准确率
- 调整匹配权重算法
- 提高匹配精度
```

## 5. 数字资产管理流

### 流程概述
数字资产是用户在平台上的核心价值体现，包括灵值、积分、数字藏品等。

### 工作流步骤

#### 5.1 查看数字资产
```
用户操作流程：
1. 进入"资产价值" → "数字资产"
2. 查看资产概览：
   - 总灵值余额
   - 灵值历史记录
   - 收益统计
   - 资产分布图表
```

#### 5.2 购买灵值
```
用户操作流程：
1. 进入"文化创作" → "购买灵值"
2. 选择充值金额：
   - 100灵值 = 10元
   - 500灵值 = 50元
   - 1000灵值 = 100元
3. 选择支付方式
4. 完成支付

系统处理：
- 创建充值订单
- 调用支付接口
- 支付成功后增加用户灵值
- 记录交易历史
- 发送充值成功通知
```

#### 5.3 灵值消费
```
灵值使用场景：
- 参与项目（扣除参与费用）
- 购买数字藏品
- 使用高级功能
- 打赏其他用户

系统处理：
- 检查余额是否充足
- 扣除相应灵值
- 记录消费明细
- 更新资产统计
```

#### 5.4 资产管理
```
用户操作流程：
1. 进入"资产价值" → "资产管理"
2. 查看资产详情：
   - 资产类型
   - 资产估值
   - 获取时间
   - 资产状态
3. 管理资产：
   - 转让
   - 质押
   - 出售
```

## 6. 分红体系工作流

### 流程概述
分红殿堂是平台的收益分配系统，用户通过参与项目、贡献资源等方式获得分红。

### 工作流步骤

#### 6.1 查看分红池
```
用户操作流程：
1. 进入"资产价值" → "分红殿堂"
2. 查看分红池信息：
   - 总分红池金额
   - 个人分红资格
   - 累计分红金额
   - 分红历史记录
```

#### 6.2 获取分红资格
```
资格获取方式：
1. 参与项目完成：每完成一个项目获得分红点数
2. 贡献资源被采用：资源被项目使用获得分红点数
3. 邀请新用户：邀请成功获得分红点数
4. 活跃度奖励：每日活跃获得分红点数

系统处理：
- 记录用户贡献行为
- 计算分红点数
- 更新分红资格等级
- 实时计算预期分红金额
```

#### 6.3 分红计算
```
分红计算公式：
个人分红 = 总分红池 × (个人分红点数 / 总分红点数) × 贡献系数

贡献系数：
- 普通贡献：1.0
- 高质量贡献：1.2
- 核心贡献：1.5

系统处理：
- 统计所有用户的分红点数
- 计算每个人的分红比例
- 应用贡献系数
- 生成分红记录
```

#### 6.4 分红发放
```
发放流程：
1. 系统定期计算分红（每月/季度）
2. 生成分红报告
3. 将分红金额转入用户账户
4. 发送分红通知
5. 记录分红历史

用户操作：
- 查看分红详情
- 查看分红历史
- 提取分红（如有提现功能）
```

#### 6.5 分红统计分析
```
用户可以查看：
- 分红趋势图
- 分红来源分析
- 贡献度排行
- 收益预测

数据展示：
- 图表化展示
- 时间维度分析
- 分类统计
```
